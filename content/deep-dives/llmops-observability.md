# LLMOps & Observability Framework

## What it is
A personal/key project where Akansh built a containerized LLM application instrumented
end-to-end for observability — a foundation for production-grade LLMOps workflows.

## What it tracks
Using **Langfuse**, the application tracks:
- model **latency**,
- **token utilization**, and
- **prompt-execution patterns**,

to support monitoring and troubleshooting of LLM behavior.

## Stack
- **Python**
- **Claude API** as the LLM.
- **Langfuse** for observability/tracing.
- **Docker** to containerize the stack for portability and reproducibility.

## Why it mattered
It reflects an operations mindset toward LLMs: not just calling a model, but measuring and
monitoring it so issues can be diagnosed in production.
